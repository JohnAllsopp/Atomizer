animation-direction

  • Instead of a smooth forward and back animation, we have an abrupt return from the end of the animation to the beginning when it recommences for a new iteration
  • There'll be times of course we may want this, but often we'll want the animation to run "backwards" to the initial state before recommencing.
  • We can use the animation-direction property to specify whether
    • the animation plays from beginning to end, before returning to the initial state and playing again, with a value of normal (the default value)
    • the animation plays in the reverse direction to how it is specified, and returning to the final values as specified before recommencing, using the value reverse
    • with a value of alternate the animation alternates in the forward and reverse direction
    • with a value of alternate-reverse the behaviour is similar to alternate but the initial animation is in the reverse direction
